python算法-链表

链表概述

链表（Linked List）是一种线性数据结构，与数组不同，链表中的元素在内存中不必连续存储。链表由一系列节点组成，每个节点包含数据域和指向下一个节点的指针。

链表的基本结构

classDiagram
    class ListNode {
        +val: int
        +next: ListNode*
        +ListNode(val)
        +ListNode(val, next)
    }
    
    class LinkedList {
        +head: ListNode*
        +size: int
        +LinkedList()
        +addAtHead(val)
        +addAtTail(val)
        +addAtIndex(index, val)
        +deleteAtIndex(index)
        +get(index)
    }
    
    ListNode "1" --> "*" ListNode : next
    LinkedList "1" --> "0..1" ListNode : head

链表的数学分析

空间复杂度分析

对于包含 $n$ 个节点的链表：

  • 基本空间复杂度：$O(n)$
  • 节点大小：$S_{node} = S_{data} + S_{pointer}$
  • 总空间：$S_{total} = n \times S_{node}$

其中：

  • $S_{data}$：数据域大小
  • $S_{pointer}$：指针域大小（通常为4或8字节）

时间复杂度对比

操作 数组 链表 数学表示
访问第i个元素 $O(1)$ $O(n)$ $T_{access}^{array} = O(1), T_{access}^{linked} = O(i)$
在头部插入 $O(n)$ $O(1)$ $T_{insert}^{array} = O(n), T_{insert}^{linked} = O(1)$
在尾部插入 $O(1)$ $O(n)$ $T_{append}^{array} = O(1), T_{append}^{linked} = O(n)$
在中间插入 $O(n)$ $O(n)$ $T_{middle}^{array} = O(n), T_{middle}^{linked} = O(i)$

总结

链表是一种基础而重要的数据结构，掌握链表的关键点：

  1. 理解指针操作：熟练掌握节点的连接和断开
  2. 边界条件处理：注意空链表、单节点等特殊情况
  3. 空间时间权衡：根据场景选择合适的数据结构
  4. 递归与迭代：灵活运用两种方法解决问题
  5. 快慢指针技巧：掌握双指针思想解决复杂问题

记住链表的核心优势：动态内存分配和高效的插入删除操作。
