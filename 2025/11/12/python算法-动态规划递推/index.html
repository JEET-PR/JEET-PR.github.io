python算法-动态规划递推

动态规划概述

动态规划（Dynamic Programming，DP）是一种通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。动态规划常常适用于有重叠子问题和最优子结构性质的问题。

核心思想

动态规划的核心思想是：

  • 分解子问题：将复杂问题分解为若干个相互重叠的子问题
  • 存储状态：保存子问题的解，避免重复计算
  • 递推求解：通过子问题的解递推得到原问题的解

动态规划的递推流程

flowchart TD
    A[开始] --> B[分析问题特征]
    B --> C{是否满足DP条件?}
    C -->|否| D[选择其他算法]
    C -->|是| E[定义状态]
    E --> F[确定状态转移方程]
    F --> G[确定边界条件]
    G --> H[编写递推代码]
    H --> I[验证结果]
    I --> J{结果正确?}
    J -->|否| K[调试优化]
    K --> H
    J -->|是| L[结束]

DP问题的判断条件

  1. 最优子结构：问题的最优解包含子问题的最优解
  2. 重叠子问题：递归算法反复求解相同的子问题
  3. 无后效性：某阶段的状态一旦确定，就不受后续决策的影响

数学基础

递推关系式

动态规划的数学基础是递推关系式，一般形式为：

$dp[i] = f(dp[i-1], dp[i-2], ..., dp[i-k])$

其中：

  • $dp[i]$ 表示第 $i$ 个状态的最优解
  • $f$ 是状态转移函数
  • $k$ 是依赖的前置状态数量

边界条件

$dp[0] = \text{初始值}$
$dp[1] = \text{初始值}$

经典问题详解

1. 斐波那契数列

问题描述：求第 $n$ 个斐波那契数

递推关系式：
$F(n) = F(n-1) + F(n-2)$

边界条件：
$F(0) = 0, F(1) = 1$

时间复杂度分析：

  • 暴力递归：$O(2^n)$
  • 动态规划：$O(n)$
  • 空间优化：$O(1)$

2. 0-1背包问题

问题描述：给定 $n$ 个物品和容量为 $W$ 的背包，物品 $i$ 的重量为 $w_i$，价值为 $v_i$，求如何选择物品使总价值最大。

状态定义：$dp[i][j]$ 表示前 $i$ 个物品在容量为 $j$ 时的最大价值

递推关系式：
$dp[i][j] = \max(dp[i-1][j], dp[i-1][j-w_i] + v_i)$

边界条件：
$dp[0][j] = 0, \forall j$
$dp[i][0] = 0, \forall i$

3. 最长递增子序列（LIS）

问题描述：给定序列，求最长递增子序列的长度

状态定义：$dp[i]$ 表示以第 $i$ 个元素结尾的最长递增子序列长度

递推关系式：
$dp[i] = \max_{j < i, a[j] < a[i]} (dp[j] + 1)$

边界条件：
$dp[i] = 1, \forall i$

动态规划的解题步骤

flowchart TD
    A[分析问题] --> B[确定状态表示]
    B --> C[找出状态转移方程]
    C --> D[确定边界条件]
    D --> E[计算顺序]
    E --> F[编写代码]
    F --> G[优化空间复杂度]

步骤详解

  1. 状态表示：明确 $dp[i]$ 的含义
  2. 状态转移：找出 $dp[i]$ 与之前状态的关系
  3. 边界初始化：确定初始状态值
  4. 计算顺序：确定状态的计算顺序
  5. 空间优化：考虑是否可以优化空间复杂度

复杂度分析

时间复杂度

动态规划的时间复杂度一般为：
$O(n \times m \times k)$

其中：

  • $n$：状态数量
  • $m$：每个状态的可能取值数量
  • $k$：状态转移的计算复杂度

空间复杂度

  • 原始版本：$O(n \times m)$
  • 滚动数组优化：$O(m)$
  • 状态压缩：$O(m)$

实战案例

编辑距离问题

问题描述：求将字符串 $s$ 转换为字符串 $t$ 的最小编辑次数

状态定义：$dp[i][j]$ 表示 $s[0:i]$ 转换为 $t[0:j]$ 的最小编辑次数

递推关系式：
$dp[i][j] = \begin{cases} dp[i-1][j-1] & \text{if } s[i-1] = t[j-1] \\ 1 + \min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) & \text{otherwise} \end{cases}$

总结

动态规划是一种强大的算法思想，掌握好动态规划需要：

  1. 多练习：通过大量练习培养DP思维
  2. 掌握模式：熟悉常见的DP模式和套路
  3. 分析能力：能够准确分析问题的性质
  4. 优化意识：时刻考虑时间和空间复杂度的优化

记住动态规划的精髓：记住过去，利用现在，规划未来。
