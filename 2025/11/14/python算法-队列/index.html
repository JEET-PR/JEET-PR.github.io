python算法-队列

队列概述

队列（Queue）是一种特殊的线性表，它只允许在表的前端（front）进行删除操作，而在表的后端（rear）进行插入操作。队列是一种先进先出（First In First Out，FIFO）的数据结构。

队列与栈的对比

特性 队列 (Queue) 栈 (Stack)
数据结构 FIFO LIFO
插入端 队尾 (rear) 栈顶 (top)
删除端 队头 (front) 栈顶 (top)
应用场景 任务调度、广度优先搜索 函数调用、深度优先搜索

队列操作的时间复杂度分析

基本操作复杂度

操作 数组实现 链表实现 数学表示
入队 (enqueue) $O(1)$ $O(1)$ $T_{enqueue} = O(1)$
出队 (dequeue) $O(1)$ $O(1)$ $T_{dequeue} = O(1)$
查看队头 (peek) $O(1)$ $O(1)$ $T_{peek} = O(1)$
判空 (is_empty) $O(1)$ $O(1)$ $T_{empty} = O(1)$

空间复杂度

对于包含 $n$ 个元素的队列：

  • 数组实现空间复杂度：$O(n)$
  • 链表实现空间复杂度：$O(n)$
  • 总空间计算：$S_{total} = n \times S_{element} + S_{overhead}$

其中：

  • $S_{element}$：单个元素的大小
  • $S_{overhead}$：指针或其他开销

高级队列算法

1. 循环队列的数学模型

循环队列的关键是模运算：

$\text{next\_index} = (\text{current\_index} + 1) \bmod \text{capacity}$

队满条件：
$\text{is\_full} = (\text{rear} + 1) \bmod \text{capacity} = \text{front}$

队空条件：
$\text{is\_empty} = \text{front} = \text{rear}$

队列大小：
$\text{size} = (\text{rear} - \text{front} + \text{capacity}) \bmod \text{capacity}$

2. 优先队列的实现

优先队列使用堆数据结构实现，时间复杂度：

  • 插入：$O(\log n)$
  • 删除最大/最小值：$O(\log n)$
  • 查看最大/最小值：$O(1)$

队列的数学性质

1. 队列的马尔可夫性质

队列系统可以用马尔可夫链来描述：

  • 到达率：$\lambda$ (单位时间内到达的顾客数)
  • 服务率：$\mu$ (单位时间内服务的顾客数)
  • 利用率：$\rho = \frac{\lambda}{\mu}$

2. Little’s Law

李特尔定律描述了队列系统中三个基本参数的关系：

$L = \lambda \times W$

其中：

  • $L$：系统中的平均顾客数
  • $\lambda$：平均到达率
  • $W$：顾客在系统中的平均等待时间

3. M/M/1 队列模型

对于 M/M/1 队列（到达和服务都服从指数分布，单个服务器）：

  • 系统中的平均顾客数：$L = \frac{\rho}{1 - \rho}$
  • 队列中的平均顾客数：$L_q = \frac{\rho^2}{1 - \rho}$
  • 系统中的平均时间：$W = \frac{1}{\mu - \lambda}$
  • 队列中的平均等待时间：$W_q = \frac{\lambda}{\mu(\mu - \lambda)}$
